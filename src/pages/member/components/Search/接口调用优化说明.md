# TreeSelect 接口调用优化说明

## 问题描述

之前的实现中，TreeSelect 的 `onChange` 事件会调用两次接口，导致不必要的网络请求和性能问题。

## 问题原因分析

经过深入分析，发现重复调用的根本原因是：

1. **双重监听机制**：

   - `handleOrgChange` 函数直接调用 `memberStore.fetchData()`
   - `useTable` hook 中的 `useEffect` 监听 `searchParams.organization_id` 变化，也会调用 `fetchData()`

2. **调用时序**：

   ```
   用户选择组织 → handleOrgChange 执行 →
   ├── memberStore.setSearchParams() (更新搜索参数)
   ├── memberStore.fetchData() (第一次调用)
   └── useTable useEffect 检测到参数变化 → fetchData() (第二次调用)
   ```

3. **架构设计问题**：
   - 数据获取逻辑分散在多个地方
   - 缺乏统一的数据流管理

## 解决方案

### 1. 统一数据流管理

**核心思想**：让 `useTable` hook 成为唯一的数据获取入口，其他地方只负责更新搜索参数。

```typescript
// 优化后的组织选择处理函数
const handleOrgChange = (value: string | undefined) => {
  console.log('Selected organization:', value);

  // 直接更新搜索参数，不触发防抖
  const newSearchParams = { ...searchParams };

  if (value === undefined || value === null || value === '') {
    delete newSearchParams.organization_id;
  } else {
    newSearchParams.organization_id = value;
  }

  // 只更新 store 中的搜索参数，让 useTable 来处理数据获取
  memberStore.setSearchParams(newSearchParams);

  // 不在这里调用 fetchData，避免与 useTable 的 useEffect 重复调用
  // memberStore.fetchData({ page: 1, size: 10 });
};
```

### 2. 数据流架构

```
用户操作 → 更新搜索参数 → useTable 监听参数变化 → 统一调用接口
```

**优势**：

- 单一数据源
- 避免重复调用
- 便于维护和调试

### 3. useTable Hook 的统一管理

`useTable` hook 中的关键逻辑：

```typescript
// 监听搜索参数变化，统一处理数据获取
useEffect(() => {
  if (searchParams.organization_id) {
    fetchData(); // 只在这里调用接口
  } else {
    console.log('No organization_id, clearing data in useTable');
  }
}, [pagination.current, pagination.pageSize, searchParams.organization_id]);

// 当组织ID变化时，重置分页到第1页
useEffect(() => {
  if (searchParams.organization_id) {
    console.log('Organization changed, resetting pagination to page 1');
    setPagination((prev) => ({
      ...prev,
      current: 1
    }));
  }
}, [searchParams.organization_id]);
```

## 优化效果

### ✅ 接口调用优化

1. **单次调用**：TreeSelect 选择组织时只调用一次接口
2. **立即响应**：不使用防抖，立即更新数据
3. **避免冲突**：与文本搜索的防抖机制分离

### ✅ 性能提升

1. **减少网络请求**：避免重复的 API 调用
2. **提升响应速度**：组织选择立即生效
3. **降低服务器负载**：减少不必要的请求

### ✅ 用户体验改善

1. **即时反馈**：选择组织后立即看到结果
2. **避免闪烁**：减少多次数据加载导致的界面闪烁
3. **操作流畅**：提升整体交互体验

## 技术细节

### 函数职责分离

```typescript
// 文本搜索 - 使用防抖
const { run: handleSearch } = useDebounceFn(
  (type: string, value: string | undefined) => {
    // 处理文本搜索逻辑
    // 适用于：姓名搜索等需要防抖的场景
  },
  { wait: 300 }
);

// 组织选择 - 立即执行
const handleOrgChange = (value: string | undefined) => {
  // 处理组织选择逻辑
  // 适用于：下拉选择等需要立即响应的场景
};
```

### 调用时机

1. **文本输入**：用户输入姓名时，使用防抖避免频繁请求
2. **组织选择**：用户选择组织时，立即更新数据
3. **默认设置**：初始化时设置默认组织，使用组织选择逻辑

## 注意事项

1. **保持一致性**：确保其他类似的选择组件也使用相同的优化策略
